What Happened

Fix #32: AutoResearch Loop Dead for 18+ Hours
The previous Cycle #5 (Fix #29) rewrote autoresearch-loop.ps1 to use Start-Process -FilePath "claude" for timeout control. This broke every subsequent run because claude on Windows is a .ps1 shim, not a Win32 .exe. Start-Process cannot execute it. 9 consecutive runs failed (Mar 18 22:30 through Mar 19 16:30), all producing the same error: "%1 is not a valid Win32 application."
Fix: Replaced Start-Process with Start-Job
Start-Job runs a background PowerShell worker that resolves claude.ps1 correctly. Timeout via Wait-Job -Timeout. Applied to both autoresearch-loop.ps1 and overnight-ralph.ps1. Verified with 3 independent tests.

Lesson Learned

Silent failures are worse than crashes
A script that exits 0 but produces no output is more dangerous than one that crashes. Crashes are visible in Task Scheduler. Silent failures look healthy. The autoresearch loop appeared "result 0" for 18 hours while doing nothing. Future scripts need output validation: if no SUMMARY line is produced, that's a failure even if the exit code is 0.
