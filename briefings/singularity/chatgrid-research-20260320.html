What ChatGrid IS

A team project knowledge management platform that lets you drop videos, docs, PDFs, web links, and text onto a visual spatial canvas ("board"), automatically vectorizes all content, and lets you or your AI coding tools query that knowledge using natural language. Integrates directly into Claude Code, Cursor, and Windsurf via MCP. Think: shared, AI-searchable knowledge layer for dev teams — between Notion, NotebookLM, and an MCP-connected RAG pipeline.
Name collision warning: Three different "ChatGrid" products exist: (1) chatgrid.ai — the knowledge board (this research), (2) ChatGrid by PNNL — a power grid visualization tool (irrelevant), (3) META by ChatGrid — a defunct messaging platform acquired by Medallia (irrelevant).

Key Capabilities (Ranked by Impact)

1. MCP Integration with Claude Code High
  • Installs in ~2 min via .mcp.json config
  • Claude Code auto-searches project boards during conversation
  • Tools: list/create boards, add content, semantic search, grounded Q&A
Could give every Claude session persistent project context that survives context resets.
2. Semantic Search + RAG Over Your Content High
  • All content chunked, embedded, stored as vectors
  • Natural language search with similarity scoring (0-1 threshold)
  • AI responses grounded in YOUR docs, not generic knowledge
  • Up to 100K characters per vectorize request
External memory layer for agents — project docs, architecture decisions, client conversations.
3. Multi-Format Content Ingestion Medium
  • YouTube videos (auto-transcript extraction)
  • PDFs, DOCX, XLSX, PPTX, CSV
  • Images (PNG, JPEG, GIF, WebP, SVG), Audio (MP3, WAV), Video (MP4)
  • Web URLs (auto-scrape + vectorize)
  • Max file size: 50 MB
One place for all project research materials, instantly searchable.
4. Visual Spatial Canvas Medium
  • Node-and-edge graph on infinite canvas
  • Nodes with x/y coordinates, metadata, connections
  • Boards can be public or private
Visual project maps. But we already have Obsidian for this.
5. Team Knowledge Sharing Medium
  • Shared API keys across team members
  • Board-level access control
  • New hires can query the knowledge base on day one
Useful for scaling. Less relevant for solo operations.
6. Full REST API Medium
  • Complete CRUD: boards, nodes, edges, chats, messages, documents, assets, jobs
  • SSE streaming for AI chat responses
  • AI model: Claude Sonnet 4
  • Rate limiting with standard headers
Fully programmable. Could integrate into autonomous tasks.

Technical Details

Architecture
  • Frontend: React SPA
  • Backend: Supabase
  • Docs: Mintlify
  • AI Model: Claude Sonnet 4
  • MCP: chatgrid-mcp npm package
  • API: REST + SSE streaming
  • Self-hosting: Not available
  • Open source: No
What's Missing
  • No Gmail, Calendar, Slack integrations
  • No voice interface
  • No mobile app
  • No self-hosting option
  • No SOC2/GDPR compliance documentation
  • No data residency controls
  • No multi-agent orchestration
  • No always-on/persistent agent features
  • Zero public reviews or track record
What to Steal From Their Approach

The per-project MCP knowledge board idea is excellent. Build a lightweight version using our existing Obsidian vault + a local vector store (ChromaDB, LanceDB, or Qdrant) exposed via a custom MCP server. Same "Claude Code auto-searches project knowledge" capability. Zero privacy risk. Zero cost. Full control.
