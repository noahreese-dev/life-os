Viral Agent Teams: Multiple Claudes Talking to Each Other

Claude Code Agent Teams (experimental) lets multiple Claude instances coordinate as a team. One lead + N teammates, each with its own 1M-token context. They communicate via shared task list + mailbox system.

  • Architecture: Lead session spawns teammates. Shared task list (pending/in-progress/completed). Direct peer-to-peer messaging. Idle notifications.
  • Communication: ~/.claude/teams/{team-name}/config.json + ~/.claude/tasks/{team-name}/
  • Models: Opus 4.6 required for lead. Teammates can use Sonnet.
  • Display modes: In-process (all in one terminal) or split-panes (tmux/iTerm2)
  • Quality gates: TeammateIdle + TaskCompleted hooks for enforcement

Life-OS implication: Combined with Desktop SSH sessions, this could enable Windows PC + Mac Mini running coordinated Claude agents. The lead on Windows orchestrates teammates on Mac Mini via SSH. This is the multi-machine dream.

Enable: set CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 in settings.json. Requires v2.1.32+. Source: Agent Teams Docs | @louisvarge tweet

The Elephant: Why We're Stuck at 52

Root Cause: Windows Task Scheduler is the Wrong Infrastructure

47 fixes over 5 days, and the system is still at 52/100. The pattern is clear:

  • Sleep/wake kills tasks — computer sleeps, Task Scheduler fires catch-up, system re-sleeps, STATUS_CONTROL_C_EXIT
  • Batch contention — all missed tasks fire simultaneously, claude-p processes compete for resources and crash
  • PowerShell encoding — UTF-8/UTF-16 mixing, em-dash parsing, here-string expansion bugs
  • Start-Process vs Start-Job vs inline — 3 different execution patterns tried, each with its own failure mode
  • Permission gates — bypassPermissions insufficient for headless runs, Ralph broken 6+ days

We have been patching symptoms. The disease is the infrastructure itself.
