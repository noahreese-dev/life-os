# Agent BTD -- Vision Document

> "What if managing your AI agents felt like playing Bloons Tower Defense?"

## What Is Agent BTD?

Agent BTD is a gamified interface layer for AI agent management. It maps the mechanics of Bloons Tower Defense -- one of the most successful tower defense franchises ever made -- onto the real work of orchestrating AI agents, shipping projects, and compounding resources.

It is not a game. It is a **productivity system that uses game design to make the work legible, engaging, and strategically clear**. The same way a BTD player reads a battlefield and makes investment decisions under pressure, a builder using Agent BTD reads their project landscape and allocates AI agents to incoming work.

The interface renders your actual work -- tasks, agents, timelines, resources -- as a living BTD map. You watch your agents pop balloons. You feel the economy grow. You make placement decisions that matter. And when a MOAB rolls in (a major milestone or crisis), you feel the weight of it and the satisfaction of your towers handling it.

## The Core Analogy

| BTD Concept | Agent BTD Mapping |
|---|---|
| **Monkeys (Towers)** | AI Agents -- each specialized, each upgradeable, each placed deliberately |
| **Balloons (Bloons)** | Tasks -- color-coded by difficulty, layered like real work |
| **The Path** | Your roadmap/timeline -- balloons travel along it, agents are placed beside it |
| **Rounds** | Sprints, phases, or time blocks -- waves of work that escalate |
| **Lives** | Deadline tolerance -- every balloon that leaks is a missed deliverable |
| **Cash** | Budget, time, energy, compute -- the resources you spend to place and upgrade agents |
| **Farms / Eco** | Automated pipelines, recurring revenue, compounding systems |
| **Heroes** | The human CEO -- limited actions per round, but uniquely powerful |
| **Maps** | Project roadmaps -- each project has its own terrain, choke points, and layout |

This is not a surface-level skin. The analogy runs deep because BTD is fundamentally a game about **resource allocation under escalating pressure** -- which is exactly what running multiple projects with AI agents is.

## The BTD Battles Eco Mechanic -- The Deepest Insight

In BTD Battles, there is a meta-mechanic that separates good players from great ones: **eco**. Instead of just defending, you can *send balloons to your opponent*. Every balloon you send generates passive income. The more you send early, the richer you are later. But if you over-invest in eco and under-invest in defense, you die.

This maps perfectly to the funded-projects-funding-vision-projects dynamic:

- **Sending balloons = Taking on funded client work.** Each project you ship generates income.
- **Eco income = Recurring revenue, compounding assets.** The $12K Boz deal, the $6K UltrawashAPP -- these aren't just one-time payments. They build relationships, portfolios, and infrastructure that pay forward.
- **Defense = Keeping your core projects alive.** IntelligenceMasters, the orb, the Life-OS system -- these don't generate revenue yet but they ARE the endgame.
- **The meta-decision every round:** How much do you invest in eco (funded work) vs. defense (vision projects)?

### Farms vs. Eco

BTD6 has two ways to generate income:

1. **Banana Farms** -- You place them, they passively generate cash every round. They cost upfront but compound over time.
2. **Eco (Battles)** -- You actively send balloons to generate income. It requires attention but scales faster.

In Agent BTD:

- **Farms = Automated agents and systems.** A Claude agent that runs nightly code reviews. A pipeline that auto-deploys. A Supabase backend that handles auth without you. These are farms. You invested once, they produce forever.
- **Eco = Manual hustle.** Taking the call. Writing the scope doc. Debugging the Expo issue yourself. Active income that requires your time and attention.

**The goal is to convert eco into farms.** Every manual process you automate is a farm placement. Every agent you train and trust is a tower that handles rounds without you. The game makes this conversion visible and satisfying.

## The Compounding Philosophy

The core financial insight BTD teaches (whether it knows it or not) is: **invest resources INTO resources so they compound.**

- A BTD player who spends all cash on attack towers survives but stays poor. A player who invests in farms early gets exponentially richer and eventually builds an unbeatable defense.
- A builder who does all the work themselves ships but stays bottlenecked. A builder who invests in agents, systems, and automation eventually runs an empire.

Agent BTD makes this visible. You can literally see:
- How many "farms" (automated systems) you have running
- What your passive "income" (automated output) is per round
- Whether you're over-indexed on manual work vs. automated systems
- The compounding curve of your total capability over time

This is the scoreboard that matters. Not "how many tasks did I complete" but "how much of my work is now automated and compounding?"

## Technological Transcendence

This project is a direct expression of the Intelligence Masters philosophy: **Technological Transcendence**.

The thesis: AI agents should handle digital life so the human can focus on in-real-life community, relationships, and presence. But right now, managing AI agents is itself a digital task. It happens in terminals, in chat windows, in dashboards that look like spreadsheets.

Agent BTD reframes this. Instead of "managing agents" feeling like IT administration, it feels like **playing a strategy game**. The cognitive mode shifts from "doing work" to "making strategic decisions." That's the unlock:

- You're not writing code. You're placing a Dart Monkey on a choke point.
- You're not triaging tickets. You're watching the balloon wave and deciding where to reinforce.
- You're not checking dashboards. You're reading the battlefield.

**The target user is the builder/CEO who spends most of their time in Claude Code and wants it to feel like playing a game.** Not because they want to avoid work -- because they want to *engage* with work at the strategic layer instead of the execution layer. The agents handle execution. You handle strategy. And strategy, rendered as a game, is genuinely fun.

This inverts the typical relationship between games and productivity. Instead of gamifying work with points and badges (which everyone sees through), Agent BTD uses an actual game's deep mechanics to make the real strategic decisions of agent management legible and engaging.

## Who This Is For

**Primary user (v1):** Noah Reese. One person, 7+ projects, multiple AI agents, running everything through Claude Code. The dashboard is for him -- a way to see his entire operation as a living game map.

**Secondary users (v2+):** Any solo builder or small team using AI agents at scale. Indie hackers, solopreneurs, AI-native startups. People who understand Claude Code, have multiple workstreams, and want a strategic layer on top of their agent operations.

**Tertiary (v3+):** Teams. Agencies. Anyone managing AI agent swarms across projects.

## The Long-Term Product Vision

Agent BTD could evolve through several forms:

1. **A Claude Code skill** -- `/btd` renders the game view of your current session. Lightweight, integrated, zero friction.
2. **A standalone local tool** -- A local web app that reads session logs, markdown files, and project state to render the full game view across all projects.
3. **A hosted product** -- A SaaS where teams can visualize their AI agent operations as BTD maps. Connect to Claude, GPT, or any agent framework.
4. **An actual game layer** -- With achievements, leaderboards, seasonal challenges. "Ship 10 projects this quarter" becomes "Survive to Round 100."

The product doesn't need to be all of these. But the design should be modular enough that each layer can be added without rewriting the foundation.

## Why BTD?

Of all possible game metaphors, BTD is uniquely suited because:

1. **Tower placement is permanent and strategic.** You don't move agents around constantly -- you place them thoughtfully and upgrade them. This maps to how you should actually manage AI agents: configure once, upgrade over time, trust the placement.
2. **The economy is the real game.** Casual players think BTD is about popping balloons. Good players know it's about resource management. Same with agent management -- the real game is resource allocation, not task completion.
3. **Escalation is built in.** Rounds get harder. Projects get more complex. The pressure curve of BTD matches the reality of growing a business.
4. **It's deeply understood.** Millions of people have played BTD. The visual language -- monkey towers, colored balloons, paths, upgrade trees -- is instantly legible. No onboarding required for the mental model.
5. **It's solo-friendly but team-capable.** BTD can be played alone or co-op. Agent BTD works for a solo builder or a team.

## The Feeling

When you open Agent BTD, you should feel what you feel when you open a BTD6 map on a hard difficulty:

- Clear about what's coming (the balloon waves are visible)
- Empowered by your towers (the agents you've built and upgraded)
- Engaged by the resource decisions (where to invest next)
- Satisfied when a big wave gets handled cleanly
- Motivated to keep building, keep upgrading, keep compounding

That feeling -- strategic engagement with escalating challenge -- is what productive work should feel like. Agent BTD makes it feel that way by making it look that way.
