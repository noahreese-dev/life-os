1. How ICS Systems Work

The ICS Ecosystem (End-to-End)

Customer arrives ↓ [Auto Passport RFID/LPR] — identifies vehicle (99.9% accuracy with AI-LPR) ↓ [Auto Sentry Terminal] — payment kiosk (EMV, NFC, cash, codes) ↓ ← CAGE API → [Card Processor / TransFirst] [WashConnect Server] — central POS + management (ASP.NET, Windows) ↓ [Tunnel Master WBC] — controls wash equipment (Ethernet + RS-485) ↓ [Gate opens, wash runs, customer exits] SEPARATELY: [WashConnect Mobile App] — operator reports (iOS/Android) [WashNOW App] — customer purchasing (iOS/Android) [Digital Menu Boards] — 46" sunlight-readable displays [Codax] — code-based access control for equipment

WashConnect Data Model

The system manages:

  • Customers: Demographics, vehicles (VIN + plate), PIN access, Cards on File
  • Club Memberships: Multi-tier, auto-billing, proration, cross-site usage, visit tracking
  • Fleet Accounts: Departments, vehicles, prepaid balances, invoicing
  • Transactions: Every wash, payment, refund, void — with journal entries
  • Promotions: Bonus points, cascade pricing, calendar triggers, frequency tracking, punch cards
  • 50+ Report Types: Financial, operational, statistical, performance analytics

3. What the $10K API Likely Includes

Estimated API Scope (Based on WashConnect Data Model + Third-Party Evidence)

  • Customer CRUD: Create, read, update customers with vehicle/plate data
  • Membership Management: Enroll, renew, suspend, terminate, auto-bill
  • Transaction Data: Wash history, payment records, revenue data
  • Wash Code Generation: Possibly — Codax codes via API
  • Fleet Operations: Account management, invoicing
  • Report Access: Financial and operational reports
  • Promotion Engine: Apply discounts, track punch cards

What it probably does NOT include:

  • CAGE API / direct payment processing access
  • Tunnel Master equipment control
  • Gate/relay direct control
  • Real-time webhooks (likely polling-only)

Industry Precedents for Bypassing POS APIs

  • Circle K + Gimbal: BLE beacons at payment terminals, authentication in milliseconds, production at scale
  • XpresWash: Bluetooth-based loyalty that works with "most existing entry systems" — no POS API needed
  • DRB Beacon Mobile: Geofence + BLE for membership redemption, no RFID tag needed
  • LoyaltyNation: QR scanning at kiosks, Apple Wallet/Google Pay, takes 10 seconds, no integration needed
  • Stamp Me: Car wash loyalty completely independent of POS system

5. Top Questions for the Call

Must-Ask (Priority Order)

  1. "What's the fee structure — one-time, annual, or per-site?"
  2. "You waived API fees for Rinsed. Is there a partner program for app developers?"
  3. "Can we review API documentation before committing to the fee?"
  4. "Is the API REST or SOAP? JSON or XML?"
  5. "Does the API support wash code generation? Can a customer purchase in-app and get a code for the Auto Sentry?"
  6. "Does WashNOW use the same third-party API, or a separate internal API?"
  7. "Does the API support webhooks for real-time events?"
  8. "Can we handle payment processing in our app (Stripe) and record the transaction in WashConnect?"
  9. "Is there a sandbox environment for development?"
  10. "Who's our technical contact for integration support?"

Information Checklist (Get Before Hanging Up)

  • Exact fee structure and payment terms
  • API documentation (or commitment to send it)
  • REST vs SOAP, JSON vs XML
  • Authentication method (OAuth, API key, JWT)
  • Sandbox/test environment availability
  • Webhook support (yes/no)
  • Named technical contact
  • Partnership program eligibility
  • Timeline from signup to first API call
  • NDA or contractual requirements
  • Whether wash code generation is available via API
  • Whether LPR/Auto Passport can be linked to third-party accounts

6. Power Moves

Negotiation Leverage

  • Rinsed precedent: "You already waive fees for partners. We're bringing the same value — more memberships, less churn, more terminal transactions."
  • Competitive pressure: DRB has Beacon Mobile built-in. FlexWash offers free API. Washify has native mobile. ICS operators are falling behind on mobile.
  • Bypass threat (use sparingly): "We can build this with BLE beacons and Codax codes without any API access. We'd prefer native integration, but the economics have to make sense."
  • Scale argument: "Every membership we sign up means more recurring revenue through your payment processing. This is a revenue driver for you, not just for us."
  • Flower Press precedent: "Flower Press has been integrating with your API for 8 years across 60+ locations. We want to bring that same capability to more operators."
